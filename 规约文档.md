# AI拍照识物与智能比价购物助手 - 需求规约文档

## 课题信息
AI 全栈比赛课题 - AI 拍照识物与智能比价购物助手

---

## 1. 背景

在日常生活中，用户常遇到"看到想买但难以检索"的问题：面对街头同款服饰、短视频里的数码产品，往往缺少品牌/款式等关键信息；即使使用截图或以图搜图，也容易出现匹配不准、结果过多、跨平台比价成本高等情况，影响决策效率。

本课题旨在通过 AI 技术打通 "识别—检索—决策" 链路，构建一款 「AI 拍照识物与智能比价购物助手」。用户完成一次拍摄后，系统可识别商品并聚合多平台推荐结果，同时支持自然语言追加筛选（如"500 元以内"），辅助用户快速收敛候选并完成决策。

项目重点在于：如何应用 VLM 视觉模型进行非结构化信息解析；如何通过 LLM 编排与 Prompt 工程实现意图理解与结构化输出等。我们希望通过技术创新，缩短用户从"产生购买意愿"到"完成消费决策"的转化路径，打造极致流畅的智能化购物体验。

---

## 2. 项目要求

实现一款面向 C 端消费者的 AI 拍照识物购物助手。客户端通过摄像头采集商品图像并上传服务端；服务端完成商品类目与关键属性（款式、颜色、品牌等）识别后，动态生成下一步决策建议卡片（如"查看同款低价"、"只看旗舰店"、"相似爆款推荐"等）。用户点击建议卡片后，系统跨平台匹配相似商品并输出推荐列表；同时支持用户通过自然语言追加筛选条件，购物助手实时刷新推荐结果，辅助用户高效完成购买决策。

---

## 3. 功能要求

### 3.1 📸 拍照识物
- 支持通过 App 内的相机拍摄商品
- 输出识别结果，包含 主要类目 与 关键属性 （品牌、颜色等）
- 基于识别关键词跨平台召回相似商品，生成推荐列表
- 支持用户对识别错误的关键属性进行 修正 （例如，将"蓝色"修正为"深蓝色"）
- 我认为还可以对店铺进行筛选，比如自营、官方旗舰店等，特别的拼多多还有品牌标识的一些店铺
### 3.2 💡 智能建议卡片
- 在识别结果下方，动态生成一组可交互的建议卡片，引导用户进入下一步购买决策
- 卡片示例："查看同款低价"、"只看官方旗舰店"、"相似风格推荐"、"查看历史价格走势"、"筛选：颜色/品牌/尺码"

### 3.3 💰 跨平台比价
- 聚合 多平台同款商品，展示各平台的最低价和平台均价
- 这里的难点在于如何准确识别商品的最低价和平台均价，以及如何处理不同平台的库存状态，可能有货和无货
- 我认为可以先根据识别结果，从多个平台查询商品信息，包括价格、库存状态等，然后根据库存状态筛选出有货的商品，最后根据价格排序，展示给用户
  
### 3.4 🗣️ 自然语言追加筛选
- 支持用户在输入框中通过 自然语言追加筛选条件
- 示例：识别出"运动鞋"后，用户可输入"帮我找 1000 元以内的黑色款，要评价 4.8 分以上的"

### 3.5 📋 推荐列表与排序
- 以卡片形式展示商品列表，每张卡片包含商品图、名称、价格、来源平台与关键标签（如"自营"、"官方"）
- 提供多种排序方式：综合推荐、价格从低到高、销量优先、好评率等

## 4. 技术架构

### 4.1 推荐技术栈

#### 客户端 (Android / iOS)
- 核心任务：摄像头图片采集、图片上传、交互界面（建议卡片、筛选器）的实现、商品列表渲染
- 技术要点：掌握原生或跨平台框架的摄像头 API、网络请求、UI 布局与状态管理（这里用flutter）

#### 服务端 (Go 和 rust)
- 核心任务：提供 API 接口（REST/GraphQL）、编排 AI 识别与检索任务、聚合多平台数据、管理数据存储与缓存
- 技术要点：掌握至少一门后端语言及主流框架，遵循设计原则，编写规范 API 接口
- 我认为可以使用Go来实现服务端，因为它们都有良好的性能和并发支持，同时也有丰富的生态系统和社区支持
- rust用来做数据隐私保护

### 4.2 技术与合规边界

✅ **允许**
- AI 能力：不限 AI 模型，不限 AI Coding 工具。可自由组合各类大模型 API 进行开发（推荐：火山引擎 OpenAPI + TRAE IDE）
- 合法数据源：使用公开合法的电商平台 API 或公开样例数据集进行比价。在缺乏合规 API 或数据来源的情况下，允许进行数据模拟，但严禁任何形式的非法数据采集
- 技术栈自由：在推荐的技术栈基础上，你可以根据团队熟悉度自由选择具体实现

🚫 **限制**
- 禁用端侧 AI SDK：为聚焦服务端架构与 AI Pipeline 的能力建设，严禁使用任何形式的端侧 AI 识别 SDK（如豆包 SDK、Google ML Kit、百度 AI SDK、阿里云视觉智能 SDK 等），所有 AI 计算须在云端完成
- 禁止非法数据采集：不得编写任何形式的网络爬虫，或以违反网站服务条款的方式获取数据

## 3.2 智能决策建议卡片
- 识别完成后动态生成多维度决策建议卡片
- 支持"查看同款低价"卡片：聚焦全网同款商品的最低价排序
- 支持"只看旗舰店"卡片：过滤出各平台官方旗舰店商品
- 支持"相似爆款推荐"卡片：推荐类目下高销量热门相似商品
- 卡片展示简洁醒目，用户一键点击即可触发对应筛选逻辑

## 3.3 多平台商品聚合
- 跨主流电商平台同步召回相似商品
- 展示商品关键信息：商品主图、标题、当前价格、平台来源、销量数据
- 支持按价格升序/降序排序
- 支持按销量排序
- 展示商品的价格波动曲线（近30天历史价格）辅助用户判断入手时机

## 3.4 自然语言交互筛选
- 支持用户用自然语言追加筛选条件，例如：
  - "500元以内"
  - "华为品牌，白色"
  - "销量过万，好评率98%以上"
  - "排除二手，要京东自营"
- LLM实时解析用户意图，自动刷新推荐结果列表
- 展示当前生效的筛选条件标签，支持点击标签快速移除

## 3.5 商品详情跳转
- 点击推荐列表中任意商品，可直接跳转至对应电商平台的商品详情页
- 保留平台原生跳转体验，不中断用户购买流程

## 4. 非功能需求

### 4.1 性能要求
- 图像上传至商品识别完成，平均响应时间 < 2秒
- 商品聚合召回完成，平均响应时间 < 3秒
- 自然语言筛选刷新结果，平均响应时间 < 1.5秒
- App首页冷启动时间 < 2秒

### 4.2 可靠性要求
- 商品识别成功率 ≥ 95%（常见日常用品）
- 服务端整体可用性 ≥ 99.5%
- 异常场景友好降级提示，不直接崩溃

### 4.3 用户体验要求
- 拍摄界面简洁，支持对焦、闪光灯控制
- 识别过程有优雅的加载动画反馈
- 整体UI风格现代轻量化，操作路径不超过3步即可触达核心功能
- 支持深色模式适配
- 一定要简单简单简单（我在想能不能加一个智能客服，或者说要不要）

### 5.3 核心技术栈
- 视觉：VLM大模型（支持多模态图文理解）
- 语言：LLM大模型 + Prompt工程，实现结构化意图输出
- 检索：向量数据库 + 关键词倒排索引，支持高效商品匹配

## 6. 选定技术栈详细方案

本项目最终采用 **Flutter + Go + Rust** 的技术组合方案：

### 6.1 Flutter 客户端方案
- 版本：Flutter 3.x Stable
- 核心依赖：
  - `camera`：相机采集与图像控制
  - `http` / `dio`：网络请求
  - `provider` / `riverpod`：状态管理
  - `image_picker`：支持从相册选择图片
  - `webview_flutter` / `url_launcher`：商品详情跳转
  - `cached_network_image`：图片缓存优化
- 优势：一套代码同时编译 Android / iOS 双端，UI 渲染性能优秀，开发效率高

### 6.2 Go 后端服务方案
- 版本：Go 1.22+
- 核心框架与依赖：
  - `gin-gonic/gin`：高性能 HTTP Web 框架
  - `gorm.io/gorm`：ORM 数据库操作
  - `redis/go-redis/v9`：Redis 缓存
  - 官方 SDK：对接火山引擎 VLM / LLM OpenAPI
  - `golang.org/x/sync`：并发任务编排（并行多平台商品召回）
- 优势：天生高并发，部署简单，非常适合编排 AI Pipeline 任务

### 6.3 Rust 安全组件方案
- 版本：Rust 1.70+
- 职责范围（数据安全相关模块）：
  - 图像上传过程中的端到端加密校验
  - 敏感配置与 API Key 的安全存储与内存脱敏
  - 用户隐私数据的加密处理
  - 高性能哈希与签名验证
- 集成方式：编译为动态链接库(.so/.dll)，Go 通过 FFI 方式调用
- 优势：内存安全、零成本抽象，适合处理安全敏感逻辑

### 9.5 GORM 数据模型定义

import (
	"database/sql/driver"
	"encoding/json"
	"time"
	"gorm.io/gorm"
)

// JSONTags 自定义JSON标签类型
type JSONTags []string
func (j JSONTags) Value() (driver.Value, error) {
	return json.Marshal(j)
}
func (j *JSONTags) Scan(value interface{}) error {
	bytes, _ := value.([]byte)
	return json.Unmarshal(bytes, j)
}

// Product 商品主表模型
type Product struct {
	ID              uint64         `gorm:"primaryKey;autoIncrement"`
	ProductID       string         `gorm:"type:varchar(64);uniqueIndex:uk_product_id;not null"`
	Title           string         `gorm:"type:varchar(255);not null"`
	ImageURL        string         `gorm:"type:varchar(512);not null"`
	Price           float64        `gorm:"type:decimal(10,2);not null"`
	Platform        string         `gorm:"type:varchar(32);index:idx_platform;not null"`
	PlatformName    string         `gorm:"type:varchar(32);not null"`
	Sales           uint           `gorm:"not null;default:0"`
	Rating          float64        `gorm:"type:decimal(3,2);not null;default:5.00"`
	DetailURL       string         `gorm:"type:varchar(512);not null"`

}

// AutoMigrate 自动建表
func AutoMigrate(db *gorm.DB) error {
	return db.AutoMigrate(
		&Product{},
		&RecognizeTask{},
		&ProductPriceHistory{},
	)
}

## 13. 术语表

### 13.1 技术术语
| 术语   | 定义                                                                |
| ------ | ------------------------------------------------------------------- |
| VLM    | Vision-Language Model，多模态视觉语言大模型，用于图像内容识别与解析 |
| LLM    | Large Language Model，大语言模型，用于自然语言意图理解与结构化输出  |
| FFI    | Foreign Function Interface，外部函数接口，Go 调用 Rust 动态库的机制 |
| TaskID | 识别任务全局唯一标识，关联图像识别结果与后续商品筛选全流程          |

### 13.2 前后端接口定义
| 接口路径               | HTTP方法 | 功能描述                                   |
| ---------------------- | -------- | ------------------------------------------ |
| `/api/v1/recognize`    | POST     | 图像识别接口：上传商品图片，返回类目与属性 |
| `/api/v1/products`     | POST     | 商品列表接口：根据筛选条件返回商品列表     |
| `/api/v1/filter`       | POST     | 自然语言筛选接口：解析用户意图并刷新列表   |

### 13.3 接口数据结构
| 结构名       | 说明                     | 关键字段                  |
| ----------- | ------------------------ | ------------------------- |
| Product     | 商品信息结构             | id, title, price, platform|
| RecognizeResult | 识别结果结构         | task_id, category, attributes, suggest_cards |
| FilterQuery | 筛选条件结构             | min_price, max_price, brand, color |

## 14. 核心业务流程描述

**完整用户主流程：**
1. 用户打开 App，进入相机页
2. 用户对准商品，点击拍摄按钮
3. 客户端压缩图像，上传至后端 `/api/v1/recognize` 接口
4. 后端调用 VLM 大模型，解析得到商品类目与结构化属性
5. 后端动态生成智能建议卡片列表，返回给客户端
6. 用户可选择：点击建议卡片 / 直接查看全部商品 / 输入自然语言追加筛选条件
7. 后端根据筛选条件，查询并聚合多平台商品列表返回
8. 用户点击任意商品卡片，跳转至对应电商平台详情页完成购买
